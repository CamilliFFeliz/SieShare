Motor de ingestão cognitiva

A demo roda 100% no GitHub Pages. A leitura real é simulada no navegador: arquivos de texto, XML e CSV são lidos parcialmente; PDFs, DOCX e XLSX são classificados por nome, extensão e contexto informado.

O que a IA verifica
  • Nome do arquivo, extensão e possíveis padrões internos.
  • Palavras-chave do conteúdo quando o navegador consegue ler o arquivo.
  • Departamento provável, pasta destino, tags e nível de confiança.
  • Padronização do nome para evitar arquivos soltos no SharePoint.
Pesquisa inteligente

A barra reinicia após cada busca, mantendo o resultado na tela. A busca considera nome, pasta, departamento, tags e texto extraído ou simulado.

Plano recomendado

IA própria + Graph API

Camada inteligente controlada pela empresa, com menor custo, maior flexibilidade e integração direta.

R$ 5,22 / mês estimado
  • Classificação antes do envio
  • Metadados customizados
  • Baixo custo por documento
  • Sem dependência de gatilhos lentos
Alternativa Microsoft

Microsoft Syntex

Bom para cenários totalmente Microsoft, mas menos atraente para esta prova por custo e lock-in.

até R$ 825,00 / mês estimado
  • Cobrança por página
  • Modelos presos ao ecossistema
  • Menor controle sobre lógica
  • Risco de custo escalar rápido
Automação operacional

Power Automate

Útil para orquestração simples, porém frágil como motor principal de classificação documental.

Latência variável
  • Depende de gatilhos/polling
  • Requer dropzones
  • Mais difícil de demonstrar em tempo real
  • Fluxo menos elegante para o usuário
Diagramas de sequência
Mais próximo do fluxo ideal: usuário envia o arquivo, back-end chama IA, recebe JSON, grava arquivo e metadados via Microsoft Graph.
Syntex: o arquivo entra primeiro no SharePoint e depois é interpretado pelo serviço Microsoft, com cobrança por página/processamento.
Automate: funciona como esteira posterior ao upload, mas depende de gatilho e pastas intermediárias, deixando a demo menos imediata.
