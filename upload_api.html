SharePoint com camada cognitiva

SharePoint Inteligente para organizar documentos automaticamente.

A aplicação transforma o upload comum em uma esteira inteligente: interpreta o documento, extrai tipo, cliente e data, cria um nome padronizado, envia para a pasta correta e permite busca por linguagem natural.

Menos arquivamento manual

O usuário não escolhe pasta manualmente. A IA interpreta o documento e sugere o destino correto.

Metadados inteligentes

Tipo, cliente, data, departamento e tags são extraídos para tornar a busca mais precisa.

Busca natural

O usuário pesquisa por termos comuns, e o sistema cruza metadados para encontrar o arquivo.

Baixo custo operacional

O modelo centralizado em API IA tende a ser mais econômico que alternativas nativas por página.

Motor de ingestão cognitiva

Nesta demo para leitura de arquivos textuais é fictícia. Para PDF, DOCX e XLSX, a tela simula a resposta esperada de uma API IA, sem gerar nomes a partir do nome original do arquivo.

O que a API IA retorna
  • Tipo documental: contrato, nota fiscal, currículo, pedido, relatório etc.
  • Cliente, fornecedor, colaborador ou entidade principal do documento.
  • Data relevante encontrada no conteúdo do arquivo.
  • Nome final padronizado sem copiar o nome original do upload.
  • Pasta final, tags e payload compatível com Microsoft Graph API.
Pesquisa inteligente

A barra reinicia após cada busca. A pesquisa consulta tipo documental, cliente, entidade, pasta, tags e texto extraído.

Automação operacional

Power Automate

Atua como orquestrador: observa uma pasta, dispara fluxo, move arquivo e atualiza metadados. Não substitui a inteligência documental.

  • Custo depende das licenças Microsoft existentes.
  • Mais latência por depender de gatilhos.
  • Requer dropzone ou biblioteca intermediária.
  • Bom complemento, mas não ideal como cérebro da solução.
Melhor uso: automações posteriores ao upload e notificações internas.
Diagramas de sequência completos
Fluxo API: upload, envio para API IA, retorno JSON, montagem da requisição, upload via Graph API, gravação no SharePoint e retorno do nome/pasta ao usuário.
Fluxo Power Automate: a IA ainda analisa o documento, mas o arquivo passa por uma pasta dropzone, aguarda gatilho e só depois é movido para a pasta final.
Fluxo Syntex: o documento entra primeiro no SharePoint, depois o Syntex processa, extrai dados e atualiza colunas/tags na biblioteca.
